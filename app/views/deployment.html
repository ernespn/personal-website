Deployment with Docker

This is the workflow that some applications follow from when a developer push a small change into the repository to when the application reaches production.

In fact. This website was deployed using it.

First we need to contanerise all the applications we want with docker.

With docker we can create images of our servers with our code shipped ready to be deployed. In this case a bunch of microservices, a front end website made in angular and bootstrap, a load balancer and even a mongoDB database

Once that is done, we need link them all toguether using docker-compose file. That will show you how the infrastructure looks like. Then we can have all our infrastructure deployed with a few commands in whatever server we want (that has docker installed).

But not everything is about deployment, you want to deploy quality images, right? Running your tests automatically and if they all pass we promote automatically that image to our image registry so it can be deployed at any time.

We use Circle CI for continuous integration which is connected to github, it detects every push we do in the repositories, run all the tests and if everything is ok, it trigger a command to create the image.

Finally, once the image is available, we can push it to production with docker cloud, where we can manage our infrastructure, redeploying images on demand (or automatically) or the full stack if we want.
