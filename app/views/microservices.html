Microservices with Load Balancing

In this example I have 5 services written in different technologies (Java, C#, F#, Javascript, Python). Each microservices has the same contract are all of them are under the same load balancer.

The angular application is making a call to the loadbalancer, binding to the view and showing the result.

Let's play with it:

The Load balancer NGINX will retry a failing request to any server.

To test this will be shuting down one of the instances and see how the request never is missing.

Go to docker terminal and run "docker stop fullstacj_node3_1"

And try again but this time look at the request. You wont find any 500 errors in the requests

Let's check a bit closer. Let's see the logs in the load balancer with "docker logs fullstacj_nginx_1".

You can find that when a request fail, it triggers automatically another one.
